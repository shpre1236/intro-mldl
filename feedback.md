# Feedback

## Overview
### 보내는 방법
- 컨C + 컨V 해서 갠톡
- 강의 끝나고 30분 내

### 마음가짐
- 서로 돕자고 하는 것이니 도움이 되는 방향으로 피드백
- 무분별한 비난 x
- 강의 관련 질문은 가능하면 수업 시간에, 가능하면 영어 피드백을 많이

## 피드백 내용

### 1. (가장 중요) 영어 피드백
- 피드백 가이드
    - 문법 틀린 것
    - 발음 틀린 내용
    - 어색한 표현 지적
- 최대한 자세하고 많이

### 2. 수업 방향성 피드백
- 피드백 가이드
    - 문제점 -> 어떻게 바꾸면 좋겠는지 순으로
        - e.g. 말이 너무 빨라서 (`문제점`) 천천히 말하면 좋겠다. (`해결책`)
    - 문제점은 가능하면 구체적으로
        - e.g. 뭐라하는지 모르겠다 (x) -> 문장이 너무 길어서 알아듣기가 어렵다 (o)

### 3. 강의 컨텐츠 관련
- 강의 내용 중에 이해 안가는 부분
- 숙제 관련 질문
- 기타 강의 때 못 물어본 내용
- 앞으로 수업 내용
- 다음 강의에 같이 공개해도 되는지 여부 같이 (기본값: 공개)

### 예시
```
1. 영어
- be 동사 빼먹음 -> The optimization process done.
- iterate on 대신에 iterate over가 적절함

2. 수업 방향성
- 글씨가 잘 안보이는데 키우면 좋겠음

3. 강의 컨텐츠
- (공개) nonlinearity가 뭐임?

```
